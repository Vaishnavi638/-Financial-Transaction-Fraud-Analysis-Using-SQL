# -Financial-Transaction-Fraud-Analysis-Using-SQL

🔍 Project Overview

This project focuses on analyzing financial transaction data to identify fraud patterns and understand risk factors using SQL.
The dataset contains ~594K synthetic transaction records representing real-world payment system behavior.

🎯 Objectives

- Analyze transaction data to identify fraud patterns
- Detect high-risk categories, customers, and merchants
- Understand how fraud varies across transaction amounts
- Identify behavioral differences between fraudulent and normal transactions

🛠️ Tools & Technologies

- SQL (MySQL)
- Data Cleaning & Transformation
- Exploratory Data Analysis (EDA)

🧹 Data Preparation

- Handled inconsistent values (e.g., ‘U’ in age column)
- Removed unwanted characters and formatting issues
- Converted data types for accurate analysis
- Ensured overall data quality and consistency

📊 Analysis Performed

- Fraud rate analysis across transaction categories
- Customer and merchant risk analysis
- Fraud distribution by transaction amount
- Fraud contribution analysis (high-risk entities)
- Behavioral analysis (fraud vs non-fraud transactions)

📌 Key Insights

- Fraud is unevenly distributed across transaction categories
- A small subset of customers and merchants contributes to a large portion of fraud
- Fraud varies across transaction amount ranges
- Fraudulent transactions deviate from normal customer behavior
- No strong time-based or rapid transaction burst patterns observed

🚀 Outcome

- Identified key fraud patterns and risk factors using SQL
- Demonstrated the use of SQL for large-scale data analysis
- Provided insights that can support fraud monitoring strategies
